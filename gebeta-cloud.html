How It Works

Every project follows the same controlled delivery flow. GitHub is the source of truth, cPanel is the runtime, and the subdomain is the public entry point. Production should only receive reviewed, committed, reproducible code.

Your Laptop
write code
→
GitHub
git push
→
cPanel
git pull
→
subdomain.gebetacloud.com
live site
ToolWhat it does
GitTracks all changes to your code locally
GitHubSource of truth for the team: branches, pull requests, review history, and release commits
cPanelProduction runtime: pulls approved code, serves static assets, or runs Node.js through Passenger
SubdomainA URL like myapp.gebetacloud.com that points to your project folder
Node.js SelectorcPanel tool that runs Node.js apps using Passenger
Production Rule

Do not deploy from a developer's laptop directly to the server. Deploy from GitHub only, so the team can review, reproduce, and roll back every release.

02

Git & SSH Setup

Do this once on every developer machine. Git records who changed what, and SSH lets each developer push securely without sharing passwords or tokens.

Install Git

Download from git-scm.com and install with default settings.

verify installation 
git --version
# Should output: git version 2.x.x

Set Your Identity

Git uses this identity on every commit. Use your real team name and company email so reviews, ownership, and rollback history are clear.

git config --global user.name "Your Name"
git config --global user.email "your@email.com"

Generate SSH Key

ssh-keygen -t ed25519 -C "your@email.com"
# Recommended for laptops: add a passphrase when prompted
# For dedicated cPanel deploy keys: use a separate key and restrict it to one repo

Copy Your Public Key

⚠ Windows Only

The cat command does not work on Windows. Use one of these instead:

# Windows Command Prompt
type %USERPROFILE%\.ssh\id_ed25519.pub

# Windows PowerShell
Get-Content ~/.ssh/id_ed25519.pub

# Open in Notepad
notepad %USERPROFILE%\.ssh\id_ed25519.pub
Your key looks like this

ssh-ed25519 AAAAC3Nza... your@email.com — copy the entire line including the beginning and your email at the end.

Remove Stored Password

If you accidentally stored your GitHub password in git config, remove it:

git config --global --unset user.password

GitHub Setup

Add your developer SSH key to GitHub so your laptop can push without passwords. For cPanel, use a separate deploy key whenever possible instead of a personal developer key.

1

Open SSH Settings

Login to github.com → click your profile picture → Settings → left sidebar → SSH and GPG keys

2

Add New SSH Key

Click New SSH key → Title: My Laptop → Key type: Authentication Key → Paste your public key → Save

3

Test the Connection

ssh -T git@github.com
# Expected: Hi YourUsername! You've successfully authenticated.
4

Push Your Project

cd your-project-folder
echo "node_modules/" >> .gitignore
echo ".env" >> .gitignore
echo "*.log" >> .gitignore
git init
git add .
git commit -m "initial commit"
git branch -M main
git remote add origin git@github.com:yourusername/your-repo.git
git push -u origin main

05

Clone Repo to cPanel

There are two supported ways to connect GitHub to cPanel. Public repos can use HTTPS directly. Private repos should use a repo-scoped deploy key first, with a short-lived token only when the cPanel GUI cannot use SSH.

Public Repo — cPanel Git Version Control GUI

For public repos this is the easiest method. cPanel pulls directly using HTTPS — no SSH key needed.

1

Open Git Version Control

cPanel → search "Git Version Control" → click Create

2

Fill In Details

FieldValue
Clone URLhttps://github.com/username/repo.git
Repository Path/home/gebetaya/myapp
Repository Namemy-repo-name
3

Click Create

cPanel clones the repo. To pull updates later: Manage → Update from Remote

Private Repo — Prefer Deploy Key

For private repos, the senior-level default is a GitHub Deploy Key. It grants this server access to one repository only and avoids sharing a developer's personal token.

1

Create a cPanel SSH Key

In cPanel Terminal, generate a dedicated key for this project. Do not reuse a developer laptop key.

ssh-keygen -t ed25519 -C "deploy-myapp@gebetacloud"
cat ~/.ssh/id_ed25519.pub
2

Add Deploy Key in GitHub

GitHub repo → Settings → Deploy keys → Add deploy key → paste the public key. Enable write access only if the server must push, which should be rare.

3

Clone with SSH

GIT_SSH_COMMAND='ssh -i ~/.ssh/id_ed25519' git clone \
  git@github.com:username/private-repo.git /home/gebetaya/myapp
Security

A Personal Access Token is a fallback, not the default. If one is required, use a fine-grained token with the minimum repository access, store it only in the deployment notes/password manager, rotate it after setup, and never paste it into screenshots, chat, or commit history.

Manual Clone via Terminal (Fallback)

# Open cPanel Terminal and run:
GIT_SSH_COMMAND='ssh -i ~/.ssh/id_ed25519' git clone \
  git@github.com:username/repo.git /home/gebetaya/myapp
06

Node.js App Setup

For projects with a server.js or app.js backend. The Node.js Selector runs your app using Passenger.

Create the Node.js App

cPanel → search "Node.js" → Create Application

FieldValueWhy
Node.js versionProject-supported LTSMatch the version used locally and documented in the repo
Application modeProductionOptimized for live servers
Application rootmyappFolder name only, not full path
Application URLSelect from dropdownLeave text box empty!
Startup fileserver.js or app.jsYour entry point file
⛔ Critical Mistake

Never type the subdomain name in the Application URL text box — only select it from the dropdown. Typing it creates a doubled folder path like myapp/myapp/ which breaks everything.

Your server.js Must Use process.env.PORT

cPanel assigns the production port through Passenger. Never hardcode port 3000 in deployed code.

import express from "express";
import path from "path";
import { fileURLToPath } from "url";

const __filename = fileURLToPath(import.meta.url);
const __dirname = path.dirname(__filename);

const app = express();
const PORT = process.env.PORT || 3000; // ← always use this

app.use(express.static(path.join(__dirname, "dist")));

app.get("*", (req, res) => {
  res.sendFile(path.join(__dirname, "dist", "index.html"));
});

app.listen(PORT, () => {
  console.log(`Server running on port ${PORT}`);
});
⚠ ES Module Warning

If your package.json has "type": "module", use import syntax not require(). Using require() will crash the app with a ReferenceError.

Install Dependencies

After creating the app, click Run NPM Install on the Node.js page, or run via terminal:

cd /home/gebetaya/myapp
npm ci --omit=dev

# If the project does not have package-lock.json yet, use:
npm install --omit=dev
Release Hygiene

Commit package-lock.json for Node.js apps. Production installs should be repeatable, especially when multiple developers are deploying the same project.

Verify .htaccess was Created

cPanel auto-generates this file. If it's missing or duplicated, restart the app.

cat /home/gebetaya/myapp/.htaccess
# Should show ONE block only:
# PassengerAppRoot "/home/gebetaya/myapp"
# PassengerBaseURI "/"
# PassengerNodejs "/home/gebetaya/nodevenv/myapp/24/bin/node"
# PassengerAppType node
# PassengerStartupFile server.js
⛔ Duplicate .htaccess Bug

If you see the Passenger block twice in .htaccess, manually fix it by running: cat > /home/gebetaya/myapp/.htaccess and paste only one block.

07

Static Site Setup

For Vite/React projects without a backend server. The server only needs the built HTML, CSS, JS, and assets.

Build Locally First

npm run build
# Creates a dist/ folder with production-ready files

Push dist Folder to GitHub

For this cPanel Git workflow, the server must receive the built dist folder. If the repo has no CI/CD pipeline that builds on the server, remove dist from .gitignore and commit the production build.

# In .gitignore — remove or comment out:
# dist

# Then push
git add .
git commit -m "build: include dist folder"
git push origin main
Team Rule

When dist is committed, rebuild it in the same pull request as the source change. A source update without matching build output can deploy old behavior.

⛔ Delete root index.html on server

If your project has an index.html in the root folder (development version with src/main.tsx), delete it from the server. It will override the correct dist/index.html and break the site.

rm /home/gebetaya/myapp/index.html

Add .htaccess for SPA Routing

cat > /home/gebetaya/myapp/dist/.htaccess << 'EOF'
DirectoryIndex index.html
RewriteEngine On
RewriteBase /
RewriteRule ^index\.html$ - [L]
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule . /index.html [L]
EOF

Point Subdomain to dist Folder

cPanel → Domains → your subdomain → Manage → Document Root:

/home/gebetaya/myapp/dist
⚠ Do NOT create a Node.js app for static sites

Static sites don't need Node.js Selector. If you accidentally created one, click Destroy. It will override your .htaccess and break the site.

08

Fix Permissions

⛔ #1 Cause of 404 on cPanel

When cPanel creates folders, permissions default to 700 (owner only). LiteSpeed web server cannot read them. Always fix permissions after setup or after cloning.

Understanding Permissions

PermissionCodeWeb Server Access
drwx------700Blocked Default — always breaks
drwxr-xr-x755Works Correct for folders
-rw-r--r--644Works Correct for files

Fix Commands — Run After Every New Deployment

# Fix the main app folder
chmod 755 /home/gebetaya/myapp

# Fix all folders recursively
find /home/gebetaya/myapp -type d -exec chmod 755 {} \;

# Fix all files recursively
find /home/gebetaya/myapp -type f -exec chmod 644 {} \;

# Fix specific key files
chmod 644 /home/gebetaya/myapp/.htaccess
chmod 644 /home/gebetaya/myapp/server.js

Restart After Fixing

# Via terminal
touch /home/gebetaya/myapp/tmp/restart.txt

# Or via cPanel: Node.js Selector → STOP → SAVE → RESTART
09

Push & Deploy Workflow

After setup, daily work should move through a pull request before production. Keep main deployable at all times.

Step 1 — Code on a feature branch

# Start from latest main
git checkout main
git pull origin main

# Create a branch for your work
git checkout -b feat/short-description

# Make your changes, then:
git status               # see what changed
git add .                # stage all changes
git commit -m "feat: describe what you changed"
git push origin feat/short-description

Step 2 — Open Pull Request

Open a pull request into main. At minimum, include what changed, how it was tested, deployment notes, and screenshots for UI changes.

Step 3 — For frontend/static projects: build before merge

npm run build            # generates dist/ folder
git add .
git commit -m "build: production build"
git push origin feat/short-description

Step 4 — Pull on cPanel after merge

Option A — cPanel GUI (Public repos):
Git Version Control → your repo → Manage → Update from Remote

Option B — Terminal (All repos):

cd /home/gebetaya/myapp
git pull origin main

Option C — Controlled overwrite (only when approved):

cd /home/gebetaya/myapp
git status
git log --oneline -5
git fetch origin
git reset --hard origin/main

Step 5 — Restart Node.js app

touch /home/gebetaya/myapp/tmp/restart.txt
# Or: Node.js Selector → Restart

Release Checklist

CheckRequired Before Deploy
Pull request reviewedAt least one teammate reviewed code or content
Build verifiednpm run build passes locally
Secrets checkedNo .env, tokens, passwords, or private keys committed
Deployment notes readySubdomain, app type, startup file, env vars, and rollback commit known
Smoke test plannedHome page, login/API path, and critical route checked after deploy

Commit Message Guide

PrefixWhen to use

fix:Fixing a bug
build:Production build / dist update

10

Private Repo Access

For private GitHub repos, use the narrowest access that works. The preferred setup is a repo-scoped Deploy Key. Use a Personal Access Token only when cPanel Git Version Control cannot pull with SSH.

Recommended: Deploy Key

1

Generate Key on cPanel

Create one SSH key per deployed project. This keeps access isolated if a single project ever needs rotation.

ssh-keygen -t ed25519 -C "deploy-myapp@gebetacloud"
cat ~/.ssh/id_ed25519.pub
2

Add Key to GitHub Repo

GitHub repo → Settings → Deploy keys → Add deploy key. Keep write access disabled unless the server has a documented reason to push.

3

Pull With SSH

cd /home/gebetaya/myapp
git remote set-url origin git@github.com:username/private-repo.git
GIT_SSH_COMMAND='ssh -i ~/.ssh/id_ed25519' git pull origin main

Fallback: Fine-Grained Token

If the cPanel GUI requires HTTPS, create a fine-grained token limited to the specific repository and read-only contents access when possible.

https://USERNAME:TOKEN@github.com/username/private-repo.git
Do Not Share Tokens in Chat

Tokens must never be sent in open chat, screenshots, commits, or documentation. Share through the approved password manager or rotate the token immediately after initial setup.

# Merge PR → Git Version Control → Update from Remote → Restart

Never edit files directly on cPanel File Manager. Always edit locally → push to GitHub → pull to cPanel. Direct edits cause merge conflicts and break deployments.

Rollback Plan

# Find previous working commit
git log --oneline -10

# Roll production back to a known good commit

Common Errors & Fixes

Every error listed here was encountered during real deployment. Read carefully.

404 Not Found

CauseFix
Folder permissions are 700chmod 755 /home/gebetaya/myapp
Wrong Application URL in Node.jsChange to correct subdomain, leave text box empty
App not runningNode.js Selector → Restart
Missing .htaccessStop app → Save → Restart to regenerate
Wrong Document RootcPanel → Domains → Manage → update path

503 Service Unavailable

CauseFix
App crashes on startupcat /home/gebetaya/myapp/stderr.log
Missing node_modulesNode.js Selector → Run NPM Install
require is not definedUse import instead of require (ES module)
Duplicate .htaccess blockManually fix .htaccess — keep only one Passenger block
Doubled folder path bugDestroy app → recreate, select URL from dropdown only

Git Pull Errors

# Error: local changes would be overwritten
git status
# If changes are server-only, document them before overwriting
git fetch origin
git reset --hard origin/main

# Error: untracked files would be overwritten
git clean -fd
git pull origin main

# Error: could not read Username (HTTPS on private repo)
# Preferred: use a GitHub Deploy Key with SSH
# Fallback: use a fine-grained token stored outside the repo

# Error: Permission denied (publickey)
# SSH key not added to GitHub account or repo deploy keys
ssh-keygen -t ed25519 -C "your@email.com"
# Add public key to GitHub → repo Settings → Deploy keys

MIME Type Error (React/Vite apps)

# Error: Expected JavaScript module but got application/octet-stream
# Cause: root index.html (dev version) is being served instead of dist/index.html
# Fix:
rm /home/gebetaya/myapp/index.html
touch /home/gebetaya/myapp/tmp/restart.txt

FileNotFoundError: doubled path .htaccess

# Error: /home/gebetaya/myapp/myapp/.htaccess not found
# Cause: Application URL text box was filled in (should be empty)
# Fix: Destroy Node.js app → recreate → leave URL text box empty
mkdir -p /home/gebetaya/myapp/myapp
touch /home/gebetaya/myapp/myapp/.htaccess

Key is Already in Use

Same SSH key cannot be added to two different GitHub accounts. Solutions:

  • Add as a Deploy Key on the specific repo (Settings → Deploy keys)
  • Use a fine-grained Personal Access Token only as a fallback
  • Make the repo public (no key needed for read access)
